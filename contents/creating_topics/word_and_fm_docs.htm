Word and FrameMaker Docs

Both Importing and Linking have been greatly simplified.

Importing Word Documents

Click File > Import and browse to the document you want to import to start the process.

You don't have to have any version of Word installed on your computer. You can import any Word document created in Word 2013 or higher and saved in DOCX format.

Initially use the default.wsf (Word Settings File) and browse to the document to be imported. As soon as you have selected the document, in the bottom right corner of the window you will see Adobe RoboHelp 2019 quickly reads it so that you can proceed with the other settings.

General Settings

Select the CSS file to be used for mapping and select the style to be used as the break point if you want separate topics based on a style.

Set the Topic Name Pattern if you want something other than the default.

 

Advanced Settings

In most cases the settings below will work best.

 

Character Settings

Your settings here will depend on your end objective.

  • If you want to retain the appearance of the Word document, set it to Import Word Style. You can click the Mapped Style dropdown to see the Word Style and you can change the mapping to another style.

  • If you want the topic to look like any other topic, select Discard Style.

See Mapping below.

Word documents retain many styles that are not used in the document, especially documents that have been worked on by many people. Previously all the style would have appeared here for mapping. In Adobe RoboHelp 2019 only the styles in use will appear.

Saving Your Settings

Before you click Import, if you have made any changes to the defaults, click Save Preset to avoid having to make those changes again. Next time you import, the last WSF file saved will appear but you can choose any other configuration you have saved.

Mapping

If you choose Discard Style as described earlier, Adobe RoboHelp 2019 will attempt to map the styles in the Word document to styles that are in the CSS you selected in the General settings.

The styles it can map will be in the topic(s) created and be much cleaner than previous versions of Adobe RoboHelp.

  • If all the styles can be mapped a CSS, the CSS you used for mapping will be applied to the topic(s) and shown in the topic properties.
    Going forward any changes to your CSS would then be applied to the imported topic(s) in the same way as all other topics.

  • If any styles cannot be mapped because they are unique to the Word document, then in topic properties you will see the CSS you used for mapping and a second CSS file . will contain definitions for the styles that could not be mapped.
    Going forward, the styles in your main CSS can be changed and will apply to the imported topic(s). The styles in the second CSS will need to be changed where they no longer work with your main CSS. You could paste them into the end of your main CSS and no longer have a link to the second CSS.

Note that in both cases, if you selected Import Inline Styles, they will be written into .

Linking Word Documents

Simply right click in the Contents Panel and select Link Word Document and browse to any DOCX file or files, you can link to more than one DOCX file at a time.


A folder will be shown in the Contents Panel with the name of the linked document. Right click the folder, select Generate HTML and your topic will be created.

Right click the folder, select Generate HTML and your topic will be created.

Printed Documentation Output

There are some big changes in the generation of printed documents, the key one being you no longer need to have Microsoft Word installed on your computer. Adobe RoboHelp 2019 generates PDF documents and ODF (Open Document Format) documents.

Creating a Printed Document Output

Click Output > Output Settings the first time you generate so that you can configure your preset as required. After the preset is configured, you can either come back here next time or you can click the Quick Generate icon top right in Adobe RoboHelp 2019.

The icons above the list of presets allow you to generate an output, view it or create further outputs.

PDF Output

The settings are quite straighforward and are described in detail in the online help.

 By default, no margins are set in the PDF options, you will need to set those.

 

 Adobe RoboHelp 2019 can create a PDF but it does not install a reader unless it is part of Adobe's Technical Communication Suite. If you don't have a reader installed, you will need to download one.

 

You cannot map to a CSS so that the styles of your document differ from the online help.

ODF Output

This feature will be supported later but these are the key points.

  • The output you create can be read in Microsoft Word as well as any word processing software that supports the ODF format. For example, Open Office and Libre Office.
  • You will not be able to map to Microsoft Word templates. Instead you will need to create a CSS file that has the styles you require.

More detail will be provided in a later version of this sample project.

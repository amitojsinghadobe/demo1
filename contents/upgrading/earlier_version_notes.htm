Earlier Version Feature Notes

The rebuild of Adobe RoboHelp from the ground up is a continuing process and some features will be incorporated in Updates as the work progresses.

In Adobe RoboHelp 2019 everything has been rewritten. The way of working has changed and some older features may not be supported. Other features will be added later. This topic give a summary and it will be revised in a later version of this sample project.

Converting WebHelp to CHMs

If you create WebHelp and then use the WebHelp to CHM script, scripting is not supported in Adobe RoboHelp 2019.

 
Command Line Generation

Command Line Generation will be added later.

 
DUCC - Dynamic User Centric Content

This was a WebHelp only feature in earlier versions and in Adobe RoboHelp 2019 it is replaced by Dynamic Content Filtering, which was a Responsive HTML5 only feature before.

 
Dynamic Content Filtering

In Adobe RoboHelp 2019, Dynamic Content Filtering is controlled by condition tags alone. Some reworking will be required if you have used build expressions.

 
External Content Search

External Content Search will not be supported.

 
FrameMaker

With every release of FrameMaker, the output has gone through many improvements and using FrameMaker to create the output directly is recommended. Creating an output from Adobe RoboHelp 2019 will be added later.

 
Index

Index keywords will be stored in topics rather than in a separate file. If you have more than one index in your project, they will be merged on upgrade. To exclude a particular keyword from a build, the method will be to apply a condition tag to it.

 
Map Files

Map files from earlier versions can be edited in a text editor. The map file editor will be added later.

 
Multiscreen

Responsive HTML5 outputs have proved much more popular and multiscreen outputs will not be continued.

 
Output Content

How you define what goes into your outputs is different in Adobe RoboHelp 2019. It is important that you understand the difference before you generate an output. It is described in The Table of Contents.

 
Printed Documentation

Output to PDF is supported in the initial release. Output to the ODF format for use with Microsoft Word, Open Office, Libre Office will be supported later. See Printed Documentation Output.

 
Project Reviews

To review your project you can create a document for review in PDF or Word supported format. There will be enhancements later.

 
Publishing

To publish your output, please generate the output locally and then upload that to the server. Direct publishing from Adobe RoboHelp 2019 including to SharePoint will be supported later.

 
Related Topics and See Also

Both these features will be made available later.

 
Resource Manager

Resource Manager has been replaced by Resource Linking. When you upgrade, any resources that you are using will be in your upgraded project and still be linked to the original file(s).

Resource Linking explains this in more detail.
RoboHelp Server

Adobe RoboHelp 2019 generates different code that require some change to Adobe RoboHelp Server. That will be supported soon.

When those changes have been made, WebHelp will also be able to be generated to Adobe RoboHelp Server so that the WebHelp Pro format will not be required. There will be no loss of functionality.

 
Source Control

GIT and Sharepoint Online will be supported in the initial release. Other source control programs will be supported later.

 
Tracking

New tracking functionality will be added later.

 
Scripting

Scripting was little used and will not be built into Adobe RoboHelp 2019.

 
XML Output

This output will not be supported in Adobe RoboHelp 2019.
